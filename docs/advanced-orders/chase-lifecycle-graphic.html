Chase Orders

A local executor keeps one passive limit order near the top of book, verifies fills before modifying, and stops safely when status is uncertain.
Client-side by design
01
Capture Context
  • Account and agent key
  • Symbol metadata and precision
  • Target size and reduce-only flag
02
Place Passive
  • Buy at best bid
  • Sell at best ask
  • Attach a deterministic cloid
03
Verify First
  • Confirm open order id
  • Reconcile fills and residual size
  • Pause on incomplete account data
04
Modify or Stop
  • Move only toward fill
  • Clear stale queued prices
  • Retry safe cancel failures
Normal Loop
Book update -> account refresh -> modify
Best bid or ask improves in the fill direction, so Kerosene queues the rounded price.
Account data confirms the order is still open and the residual size is current.
The same order id is modified at the verified residual size.
Ambiguous Status
Fail closed, then reconcile
Unknown placement responses are checked by cloid before another place can happen.
Missing or incomplete account snapshots pause replacement instead of guessing.
Stop Path
Cancel the live order id
Stop requests wait for in-flight place or modify results before cancelling.
Retryable cancel failures are rearmed on the app status tick, bounded by the retry cap.
